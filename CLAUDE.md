# Калькулятор рентабельности майнинга BTC

Standalone HTML-приложение для расчёта прибыльности майнинговой фермы с учётом особых условий оплаты (% от добычи + рубли за электричество).

## Архитектура

Один файл — `mining_calculator.html`. Внутри:
- HTML структура (3 вкладки)
- CSS (тёмная тема, оранжевый акцент)
- JavaScript (vanilla, без фреймворков)
- Chart.js через CDN для графиков
- localStorage для журнала записей

Решение быть в одном файле — приоритет портативности (открывается любым браузером без сервера). При желании можно разделить на `index.html` + `styles.css` + `app.js` для удобства поддержки.

## Три раздела

1. **Классический расчёт** (`#tab-classic`) — стандартный майнинг: платишь рублями, получаешь BTC. Считает прибыль, точку безубытка, маржу.

2. **Особые условия** (`#tab-special`) — гибрид: часть оплаты в рублях, часть как % от добытого BTC. Сравнивается с фиксом 0.02046 BTC/мес который был в исходном договоре.

3. **Журнал** (`#tab-journal`) — реальные данные по месяцам с CRUD-формой, кастомным month-picker, автоподгрузкой исторической сложности, mark-to-market PnL для HODL-позиции и "что-если" калькулятором.

## Live данные (top banner)

Подгружаются автоматически на старте, можно обновить:
- **Цена BTC** ($USD): jsDelivr CDN (приоритет, работает из file://), CoinGecko, Binance, Coinbase, Bitstamp, Kraken, blockchain.info + прокси
- **Курс USD/RUB**: ЦБ РФ (cbr-xml-daily.ru), jsDelivr CDN, open.er-api.com + прокси
- **Сложность сети** (T): blockchain.info, mempool.space, blockstream.info, btc.com + прокси

CORS-прокси для fallback при открытии через file://: api.codetabs.com, api.allorigins.win, proxy.corsfix.com.

## Историческая сложность (для записей в журнал)

При выборе месяца автоматически фетчится сложность сети на 28-е число этого месяца:
- mempool.space `/api/v1/mining/blocks/timestamp/{ts}` → высота блока
- Потом `/api/v1/blocks/{height}` → difficulty
- Кешируется в localStorage под ключом `btc_diff_cache_v1`

## Формулы расчёта

Награда блока **3.125 BTC** (после халвинга апрель 2024).

```
Добыча BTC = hashrate × 86400 × 30 / (difficulty × 2^32) × 3.125
```

Mark-to-market PnL для HODL:
```
cost_basis = total_spent_usd / total_btc_mined
hodl_value_now = hodl_btc × current_btc_price
unrealized_pnl = hodl_value_now - (hodl_btc × cost_basis)
```

## Хранение данных

localStorage keys:
- `mining_journal_v1` — массив записей журнала
- `btc_diff_cache_v1` — кеш исторической сложности по месяцам (`{"2026-05": 138.6, ...}`)

Структура записи журнала:
```js
{
  id: timestamp,
  month: "YYYY-MM",
  kwh: number,
  rubElec: number,        // ₽ заплачено за электричество
  hashrate: number,       // TH/s средний за месяц
  difficulty: number,     // T (опциональное)
  btcMined: number,       // BTC намайнено
  btcSold: number,        // BTC продано
  salePrice: number,      // $ цена продажи
  usdRub: number          // курс RUB при продаже
}
```

## Известные ограничения / TODO

- При открытии через `file://` некоторые API блокируются CORS — частично решено через CDN-источники и прокси, но не 100%
- Нет JSON-импорта журнала (только CSV-экспорт)
- Расчёт себестоимости 1 BTC — простой average, не учитывает FIFO/LIFO при продажах
- Нет учёта pool fee, uptime, прочих операционных расходов в журнале (только в классическом расчёте)
- Mobile-вёрстка не идеальна для таблицы истории (горизонтальный скролл)

## Возможные улучшения

- Добавить в журнал поля: pool fee %, uptime %, прочие расходы ₽ (обслуга/ремонт/хостинг)
- График hashprice ($/TH/day) — индустриальный бенчмарк
- Realization rate: фактическая добыча vs теоретическая по формуле — индикатор операционного здоровья
- JSON импорт/экспорт всего журнала
- Графики на главной странице с прогнозом окупаемости
- Расщепить файл на html/css/js для удобства редактирования

## Тех-детали для разработки

- Vanilla JS, без сборки. Откроется в любом современном браузере.
- Chart.js 4.4.1 через CDN
- Поддержка fallback при отсутствии интернета (всё кроме live-данных работает офлайн)
- Тёмная цветовая схема (CSS variables в `:root`)
- Шрифт: системный sans-serif (`-apple-system, Segoe UI, system-ui`)
- Все строки на русском

## Запуск локально

Просто открыть `mining_calculator.html` двойным кликом — работает.

Если хочешь чтобы все live-источники работали без CORS-ограничений (которые бывают при `file://`), запусти локальный сервер:

```bash
python3 -m http.server 8080
```

И открой `http://localhost:8080/mining_calculator.html`
