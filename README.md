# ₿ Калькулятор рентабельности майнинга BTC

Standalone веб-калькулятор для расчёта прибыльности майнинговой фермы и ведения журнала реальных данных по месяцам. Одностраничное приложение без бэкенда — все данные хранятся в `localStorage` твоего браузера.

## Возможности

- **Классический расчёт** — стандартный майнинг (рубли за электричество → доход в BTC). Считает прибыль, точку безубытка, маржу.
- **Особые условия** — гибридная оплата: часть в рублях за электричество, часть как % от добытого BTC. Сравнение с фиксированной ставкой.
- **Журнал** — реальные данные по месяцам с CRUD, графиками, mark-to-market PnL для HODL-позиции и «что-если» калькулятором.
- **Multi-user** — два независимых журнала (D и G) с режимом сравнения D+G.
- **Live данные** — автоматически подтягиваются цена BTC, курс USD/RUB и сложность сети с десятка источников (CDN, биржи, mempool.space, ЦБ РФ) с fallback-цепочкой.
- **Историческая сложность сети** — при выборе месяца в журнале автоматически фетчится сложность на 28-е число.

## Как пользоваться

1. Открой [сайт](#) (ссылка на GitHub Pages — добавь после деплоя).
2. Сверху видишь live-данные: цена BTC, курс рубля, сложность сети. Клик по карточке → применить в общие параметры.
3. Переключайся между тремя вкладками: классический расчёт, особые условия, журнал.

## Синхронизация и приватность данных

Данные журнала **зашифрованы AES-256** паролем и сохраняются в облако через GitHub (файл `data/journal.json` в этом репозитории).

- ничего не теряется, даже если очистишь cookies или сменишь устройство
- ты и партнёр видите изменения друг друга
- история всех изменений в git log
- **без пароля файл в репо — бессмысленный мусор**, никто не прочтёт даже зайдя по прямой ссылке
- пароль никогда не уходит на сервер — он живёт только в твоём браузере
- localStorage остаётся офлайн-кешем, приложение работает и без интернета

Кнопка **☁️ Sync** в журнале показывает состояние:
- 🟢 **Синхронизировано** — всё ок, запись доступна
- 🔒 **Заблокировано** — нужен пароль для расшифровки
- 🔵 **Только чтение** — PAT не настроен, видишь данные, но не можешь записать новые
- 🟡 **Синхронизация…** — идёт обмен с GitHub
- 🔴 **Ошибка** — клик по индикатору покажет детали
- ⚫ **Sync не настроен** — выключен или не настраивался

### Первая настройка (владелец, один раз)

1. Открой [сайт](https://kibertec.github.io/mining-calculator/) → вкладка **Журнал**.
2. Клик по **☁️ Sync** → секция «🔐 Пароль шифрования» → **Задать пароль**.
3. Придумай надёжный пароль (8+ символов). **Запиши в надёжное место** — восстановить невозможно.
4. Создай **Personal Access Token (PAT)** для записи в GitHub:
   - Открой https://github.com/settings/tokens?type=beta (Fine-grained)
   - **Generate new token** → имя любое, expiration максимум или «No expiration»
   - **Repository access** → **Only select repositories** → `KiberTec/mining-calculator`
   - **Permissions** → **Repository permissions** → **Contents: Read and write**
   - **Generate** → скопируй (`github_pat_...`)
5. Вставь токен в поле PAT в той же модалке Sync → **Сохранить**.

С этого момента все изменения автоматически шифруются и кладутся в облако.

### Подключение партнёра

1. Пригласи партнёра как collaborator в репо: открой https://github.com/KiberTec/mining-calculator/settings/access → **Add people** → его GitHub username.
2. Партнёр принимает приглашение по email.
3. Партнёр заходит на [сайт](https://kibertec.github.io/mining-calculator/), переходит в Журнал — будет lock-screen «Журнал заблокирован».
4. **Передай партнёру пароль лично** (Telegram/звонок/etc — не публично).
5. Партнёр вводит пароль → видит твои данные.
6. Если партнёру нужно вносить свои записи — пусть создаёт свой PAT по той же инструкции.

Если кто-то посторонний случайно зайдёт по ссылке — увидит сайт и заблокированный журнал. Без пароля внутрь — никак.

## Запуск локально

Просто открой `index.html` двойным кликом — работает в любом современном браузере.

Если хочешь чтобы live-источники работали без CORS-ограничений (некоторые блокируются при `file://`):

## Технические детали

- **Vanilla JS**, без сборки и фреймворков. Один HTML-файл с inline CSS и JS.
- **Chart.js 4.4.1** через CDN — единственная зависимость.
- **GitHub Contents API** для облачной синхронизации (read без auth, write с PAT).
- **Шифрование**:
  - Алгоритм: AES-GCM-256 (Web Crypto API)
  - KDF: PBKDF2 SHA-256, 200 000 итераций
  - Salt 16 байт + IV 12 байт генерируются случайно при каждом сохранении
  - Пароль никогда не сериализуется и не отправляется на сервер
- **localStorage** для офлайн-кеша:
  - `mining_journal_D`, `mining_journal_G` — записи по пользователям (расшифрованные)
  - `mining_journal_user_v1` — последний активный пользователь
  - `btc_diff_cache_v1` — кеш исторической сложности по месяцам
  - `mining_gh_token_v1` — PAT для записи в GitHub
  - `mining_gh_sha_v1` — SHA текущей версии файла данных
  - `mining_crypto_pass_v1` — пароль шифрования (если выбран «Запомнить»)
- **Стратегия merge** при параллельных правках: для каждого месяца у каждого пользователя оставляется запись с большим `id` (timestamp создания/последнего редактирования).

## Формулы

Награда за блок: **3.125 BTC** (после халвинга апрель 2024).

```
Добыча BTC = hashrate × 86400 × 30 / (difficulty × 2^32) × 3.125
```

Mark-to-market PnL для HODL-позиции:

```
cost_basis    = total_spent_usd / total_btc_mined
hodl_value    = hodl_btc × current_btc_price
unrealized    = hodl_value − (hodl_btc × cost_basis)
```
